- Keep scheduled-job execution behind `backend/jobs/mock-reminder-job-runner.mjs`; server/routes should trigger job-runner APIs only.
- Keep database readiness and production gap reporting behind `backend/services/mock-database-service.mjs`; server/routes should not embed database driver or migration logic.
- Keep frontend database status display wired to `GET /api/database/status` so production-database progress remains visible before the real adapter is connected.

- Keep audit retention/redaction behind `backend/services/mock-audit-service.mjs`; repository audit writes should keep redacting sensitive metadata before persistence.
