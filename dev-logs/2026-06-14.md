## 问题风险
- localhost.run 免费临时域名会频繁轮换，旧链接失效是预期风险，不代表 App 本体坏了。
- 如果要稳定分享给外部用户，需要实际创建 Render 或同类托管服务并填写运行时密钥；当前只是仓库内蓝图和启动配置。
- 守护脚本已具备自动重启 SSH 隧道能力，但它仍依赖本机在线、电脑不休眠、网络稳定和 `localhost.run` 可用；正式外部演示仍建议固定线上测试环境。
- 当前执行环境不能可靠持久化后台守护进程；本地守护需要保持前台会话运行，长期稳定访问应迁移到 Render 或同类托管平台。
- 完整真实 AI 仍取决于 provider 额度、速率限制和结构化输出质量；v72 解决的是失败可解释、冷却可见和安全改写，而不是绕过 provider 限制。
- localhost.run 仍是免费临时隧道，即使 watchdog 和 curl 兜底已增强，也不能等同固定线上环境；正式演示仍应创建 Render 或同类托管服务。
- 当前 shell 没有 `npm` 命令时，可用 `node scripts/public-preview-status.mjs` 直接读取公网状态。
- `npm run check:stable-hosting` 只是上线前预检；只有真正创建 Render 或同类托管服务并通过固定 URL 连续验收后，才能替代临时 `lhr.life` 链接。
- 临时隧道仍存在 DNS/解析不稳定：同一时段直接 `curl` 当前地址可返回 `200`，但健康检查子进程可能遇到短暂 DNS resolve 失败。固定托管仍是 P0 根治方向。
- Watchdog 的连续失败阈值只能降低误杀，不能把 `localhost.run` 变成稳定生产入口；固定线上测试环境仍是 P0 根治方案。
- 本机备用地址只能在本机访问，不能替代固定公网部署；它的价值是隧道失败时让本机继续演示和排查。
- 访问报告能快速判断当前可用入口，但仍不能替代固定托管；如果推荐结果为 `local-fallback`，外部用户仍打不开公网链接。
- 本次 3 分钟验收只能证明当前临时隧道在该窗口内稳定，不代表 `lhr.life` 后续不会轮换或掉线；固定托管仍是 P0 根治方向。
