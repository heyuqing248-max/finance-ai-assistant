- 提醒规则、通知中心这类账号异步操作必须在请求发起时记录会话版本；退出登录或账号变化后，旧响应必须返回 `null` 并且不得更新页面成功状态。
- 自选股读取、新增、删除也属于账号异步操作，必须使用 `activeWatchlistRequestId` 和 token/user 校验阻止旧响应污染当前页面。
- 偏好读取/保存属于账号异步操作，必须使用 `activePreferencesRequestId` 和 token/user 校验阻止旧响应污染当前分析风格、提醒渠道和状态提示。
- 登录流程也必须使用 `activeAccountRequestId`；退出登录后，未完成的登录流程不得继续显示登录成功提示或继续串联后续账号同步。
- 合规确认与适当性问卷属于账号异步操作，必须使用 `activeComplianceRequestId` 和 token/user 校验阻止旧响应写入风险确认、问卷记录或覆盖状态提示。
- AI 分析请求必须使用 `activeAnalysisRequestId`、股票代码、风险偏好和 token/user 校验；退出登录后旧账号分析、行情补充和历史保存响应不得渲染或提示。
- 新闻加载请求必须同时记录市场和股票代码；返回时如果市场或股票已变化，旧新闻、个股情报、公告和公开言论不得渲染。
- 持仓读取/保存属于账号异步操作；退出登录时必须递增 `activePortfolioSyncRequestId`，旧账号持仓响应不得写入当前页面、不得标记已同步、不得覆盖退出登录提示。
- 账号角色管理属于账号异步操作；角色切换、管理员授权、管理员撤销、角色历史刷新必须记录 `activeAccountRequestId`、token 和用户 id，返回时如账号状态变化必须忽略旧响应。
- 调度维护属于账号异步操作；死信队列刷新、死信任务重放、worker 健康刷新、worker 心跳记录、调度队列刷新、样例任务创建、到期任务处理和 worker nonce 清理必须记录 `activeSchedulerMaintenanceRequestId` 以及账号上下文，退出登录或账号变化后旧响应不得更新调度面板、缓存或成功/失败提示。
- 审计维护属于账号异步操作；审计保留清理、证据包生成、校验、归档、回执刷新、下载交接和回放预演必须记录 `activeAuditMaintenanceRequestId` 以及账号上下文，退出登录或账号变化后旧响应不得更新审计面板、Hash 链摘要、下载动作或成功/失败提示。
- Demo token 获取属于账号敏感操作；`ensureDemoAuthToken()` 必须记录 `activeAccountRequestId`，退出登录或账号变化后，迟到 token 响应不得写入 auth state 或 localStorage。
- 自动获取 demo token 后才能继续的维护操作，必须通过 `resolveFreshAuthToken()` 停止旧账号链路；如果返回空 token，调用方必须直接返回 `null`，不得继续发后端维护请求或渲染错误状态。
- 通知中心读取、标记已读和投递重试也属于账号异步操作；自动获取 demo token 后必须用 `createAccountRequestContext()`/`isAccountRequestStale()` 判断真实账号变化，不能把本次新 token 误判为旧响应。
- 持仓读取/保存属于账号异步操作；自动获取 demo token 后也必须通过 `resolveFreshAuthToken()` 与账号上下文校验，旧账号链路不得继续请求 `/api/portfolio` 或更新持仓同步状态。
- AI 分析历史保存属于分析后的账号异步操作；如果 `resolveFreshAuthToken()` 返回空 token，必须直接停止历史保存，不得请求 `/api/analysis/history`，但当前已生成分析可以继续保留。
- 偏好读取/保存属于账号异步操作；自动获取 demo token 后必须复核账号状态，旧账号链路不得请求 `/api/preferences` 或覆盖当前风险偏好/提醒偏好。
- 自选股读取/加入/移除属于账号异步操作；自动获取 demo token 后必须通过 `resolveFreshAuthToken()` 停止旧账号链路，不得请求 `/api/watchlist` 或覆盖后端同步提示。
- 提醒规则读取/加入/移除属于账号异步操作；自动获取 demo token 后必须通过 `resolveFreshAuthToken()` 停止旧账号链路，不得请求 `/api/reminders` 或覆盖后端同步提示。
- 合规风险确认和适当性问卷属于高敏感账号异步操作；自动获取 demo token 后必须通过 `resolveFreshAuthToken()` 停止旧账号链路，不得请求合规保存/读取 API 或覆盖退出登录后的风险确认状态。
- 真实 AI 模型接入不得只依赖 provider/API key/合规复核；还必须有成本预算、每用户限流、token 上限、来源覆盖和证据不足输出不确定的门禁。运行时仍必须保持 `runtimeMode: inactive`，直到人工审批和 smoke test 完成。
- 本地开发环境里，如果后端已连接，页面启动也必须刷新后端服务状态，避免旧 localStorage 缓存掩盖新后端门禁。
- 真实 AI 模型调用预检只能先做 dry-run；即使门禁配置齐全，`canExecuteLiveCall` 和 provider adapter 的真实调用能力仍必须保持关闭，直到人工审批和运行时开关完成。
- AI 模型审计 envelope 不得包含原始 prompt、原始模型响应、API key、原始来源全文、原始持仓备注、联系方式或任何可直接恢复用户敏感信息的字段。
- 生产认证也必须先做 dry-run 预检；即使 provider、MFA、邮箱验证、审计、隐私、登录风控和账号恢复门禁全部 ready，`canExecuteProductionAuth` 仍保持 false，直到人工审批和运行时切换。
- 认证审计不得记录明文密码、密码哈希、refresh token、MFA secret 或原始设备指纹；账号恢复不得支持仅靠安全问题或静默关闭 MFA 的流程。
- 外部通知投递也必须先做 dry-run 预检；即使 provider、授权、回执、退订抑制、反弹处理、审计和隐私门禁全部 ready，`canExecuteExternalDelivery` 仍保持 false，直到人工审批和运行时切换。
- 通知投递审计不得记录原始消息正文、邮箱、手机号、微信 OpenID 或 Telegram chat id；退订、硬反弹、投诉和隐私删除用户必须优先进入 suppression，避免静默继续外投。
- 后台 worker 调度也必须先做 dry-run 预检；即使 provider、DLQ、worker 健康、背压、回调认证、runbook 和审计门禁全部 ready，`canStartBackgroundWorkers` 仍保持 false，直到人工审批和运行时切换。
- 调度审计不得记录 worker secret、cron signing secret、raw signature 或 raw payload；队列背压和 runbook 未确认前，不得启动真实 cron、队列 worker 或后台网络任务。
- 行情 provider 也必须先做 dry-run 预检；即使 provider、API key、授权分层、延迟标签、缓存、限流、署名和预检门禁全部 ready，`canRequestProvider` 仍保持 false，直到人工审批和运行时切换。
- 行情审计不得记录 raw tick、完整 order book、provider API key、交易所用户协议全文或交易指令；价格和图表附近必须保留 sample/delayed/live 延迟标签。
- 新闻/公告/公开言论 provider 也必须先做 dry-run 预检；即使 provider、授权、来源验证、再分发、限流和预检门禁全部 ready，`canFetchProviderContent` 仍保持 false，直到人工审批和运行时切换。

- 宏观审计不得记录 raw policy document、provider API key、calendar session cookie 或交易指令；宏观评分旁必须保留 asOf 和新鲜度标签，过期指标不得标注为当前官方宏观数据。
