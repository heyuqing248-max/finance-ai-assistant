## 问题与风险 / Issues and Risks
- 真实生产数据库、真实行情/新闻源、真实 AI 服务、真实邮件/微信/Telegram 推送仍未接入。
- Xcode/iOS 封装不再作为当前交付目标；历史 wrapper 文件保留，但后续不应默认用 Xcode 构建证明网页完成度。
- 切换审计链目前是 metadata-only 证据计划，不会写真实审计记录，也不会读取生产审计事件。
- 当前会话没有可调用的浏览器自动化工具，因此本轮没有截图级网页视觉验收。
- 当前环境没有 `npm` 命令，因此无法直接执行 `npm run dev:web`；已用 `node scripts/web-dev-server.mjs` 验证同一个网页服务器脚本。
- 一键启动脚本仍依赖本机 Node；当前环境无法验证 `npm run dev`，但会直接验证 `node scripts/full-dev-server.mjs`。
- 进程停止时当前工具会话没有可写 stdin，因此本轮使用精确 `pkill` 释放测试端口；后续长跑服务建议用可交互终端启动。
- 自动连接只在本地开发网页地址启用，正式部署网页不应默认连接用户电脑的 localhost。
- 当前 PWA 离线能力用于打开已缓存的网页壳和本机样例数据；真实行情、新闻、AI 和账户同步仍依赖后端或未来真实服务。
- 应用壳资源在线时应优先取网络，避免频繁迭代阶段用户被旧缓存卡住；普通图片/字体等静态资源仍可运行时缓存。
- 浏览器是否显示安装入口取决于系统和浏览器安装规则；当前网页只在浏览器发出安装事件后显示“安装到设备”按钮。
- 最近搜索目前仅保存在本机浏览器；后续如要账号同步搜索历史，需要纳入隐私设置和用户授权。
- 搜索输入进入页面展示时必须继续使用 `escapeHtml` 或结构化 DOM API，后续不要把用户输入直接拼进 HTML 字符串。
- 目前搜索历史没有账号同步和服务端存储；后续若接入账号同步，需要增加明确授权、隐私说明和删除/导出能力。
- 搜索框为空提交时必须清理 `lastSearch`，避免刷新后恢复用户已经主动清空的旧关键词。
- 市场 tab 切换到默认样例股票时必须清理搜索框和 `lastSearch`，避免搜索输入和当前股票展示不一致。
- 自选股点击切换股票时也必须清理搜索框和 `lastSearch`；默认样例切换、自选股切换、空搜索都共享 `clearSearchInputState()`。
- 启动时恢复 `lastSearch` 前必须确认该关键词匹配当前选中股票；错配时清除旧搜索词，匹配时才保留输入框内容。
- 新闻情报保存/刷新这类异步请求必须记录发起时的市场和股票代码；返回时如果当前股票已变化，必须忽略旧结果，不得写入当前股票页面或本机持久记录。
- 持仓保存/读取这类异步请求同样必须记录发起时的股票代码；保存请求还必须固定 payload，返回时如果当前股票已变化，必须忽略旧结果。
- 通知中心读取请求必须记录账号会话版本和 auth token；退出登录或不可同步状态会递增版本并清空通知，旧账号响应不得渲染到已退出页面。
- 通知标记已读、失败投递重试等账号异步操作也必须记录账号会话版本和 auth token；退出登录或账号变化后，旧响应不得渲染成功提示、恢复旧通知列表或触发旧账号刷新。
