## Remaining Issues / 剩余问题
- **P2: Real notification delivery still needs backend/native follow-up.** The current prototype now explains per-channel availability, stores concrete reminder rules locally, syncs them to the mock backend, can manually evaluate triggers, run a mock job path, create mock notification records, track delivery attempts, and retry failed mock deliveries, but real scheduled delivery workers and real push/email/WeChat/Telegram channels are not implemented yet.
- **P2: Chinese text input was not fully verified through the simulator hardware keyboard.** English stock search works; Chinese search should be retested with a configured Chinese input method or native search picker.
- **Expected MVP gap: Real data, login, backend, notifications, and live AI services are not connected yet.** Current values and trend charts are sample data only.
- **Expected integration gap: Core frontend flows can now read/write through the mock backend, including preferences, analysis history, queued notification records, and mock email-auth sessions, and the mock backend now has provider/repository/database/audit/AI-service/auth-service/notification-service boundaries plus optional JSON persistence and visible repository/database/audit status, but production services are still not real.** Real providers, production database, production auth provider, tamper-evident audit storage, real AI model integration, encryption, retention policy, and scheduled notification delivery remain future backend work.
- **Expected database gap: Migration dry-run is no-write planning only.** It proves the current table order and blockers are visible before a real database adapter is connected, but it does not create schemas, migrate data, or verify a live Postgres/Supabase connection yet.
- **Expected SQL gap: SQL draft generation is review-only.** It creates deterministic PostgreSQL draft statements for future migration tooling, but does not run migrations, manage versions, execute rollback, or verify a live database.
- **Expected migration-package gap: Migration package is release-readiness metadata only.** It version-tracks the current schema draft and gates, but does not execute migration SQL or replace a real migration tool.
- **Expected read-only health gap: Read-only connection health is readiness metadata only.** It reports config/driver blockers and read-only guardrails, but does not yet open a real database connection or verify live Postgres/Supabase.
- **Expected news-intelligence persistence gap: Mock persistence is now available, but production still needs real migrations, licensed data retention rules, source-attribution policy, and human-review workflows before it can become a production intelligence store.**
- **Expected news-ingestion runtime gap: News ingestion telemetry now covers sample dedupe, attribution gates, cooldown windows, and no-social-scraping safety flags, but production still needs licensed providers, durable dedupe storage, retention policy, and legal review before live news or public-statement monitoring can be enabled.**
